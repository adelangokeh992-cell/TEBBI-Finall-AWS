# قائمة BAA (Business Associate Agreement) — TEBBI

للامتثال لـ HIPAA، يجب توقيع **اتفاقية Business Associate (BAA)** مع كل طرف يتعامل مع PHI نيابة عنك.

## أطراف يُنصح بإبرام BAA معها

| الطرف | نوع الخدمة | ملاحظات |
|-------|------------|----------|
| مزود قاعدة البيانات (مثلاً MongoDB Atlas، AWS DocumentDB) | استضافة بيانات تحتوي PHI | تأكد من أن الخدمة تقدم BAA وتوقيعها |
| مزود الاستضافة (السيرفرات، الحاويات، Kubernetes) | تشغيل التطبيق وقاعدة البيانات | AWS, GCP, Azure توفر BAA لخدمات معينة |
| خدمة البريد (إرسال إشعارات، روابط) | إن كانت تلمس محتوى قد يحتوي PHI أو معرفات | مراجعة سياسة الخدمة وطلب BAA إن لزم |
| خدمة النسخ الاحتياطي (S3، تخزين سحابي ثانٍ) | تخزين نسخ احتياطية قد تحتوي PHI | BAA مع مزود التخزين |
| OpenAI أو أي معالجة NLP/ذكاء اصطناعي على نصوص طبية | إن كان يُرسَل نصوص تحتوي PHI | مراجعة الشروط واتفاقيات الاستخدام وطلب BAA إن أمكن |
| Google (Gemini) | نفس الاستخدام كـ OpenAI في التطبيق | عند تفعيل مفتاح Gemini وإرسال نصوص/صور تحتوي PHI، يُنصح بتوقيع BAA مع Google |

**ملاحظة:** عند تفعيل الذكاء الاصطناعي (OpenAI أو Gemini) من إعدادات العيادة مع بيانات مرضى حقيقية، يُوصى بشدة بوجود BAA موقّع مع المزود قبل الإرسال. يمكن تذكير المستخدمين بذلك في واجهة الإعدادات أو في التوثيق.

## الإجراء

1. تحديد كل الأطراف التي تلمس PHI (من [خريطة بيانات PHI](phi-data-map.md) ومن البنية التحتية).
2. طلب BAA من كل مزود خدمة معتمد لديه إمكانية الوصول إلى PHI.
3. الاحتفاظ بنسخ موثّقة من الاتفاقيات الموقعة وتواريخها.
4. مراجعة القائمة سنوياً عند تجديد الخدمات أو إضافة تكاملات جديدة.
