# سجلات التدقيق (Audit Logs) — TEBBI

## نظرة عامة

يُسجّل النظام أحداث الوصول والتعديل على البيانات الصحية والموارد الحساسة في مجموعة `audit_logs` في MongoDB. الوصول إلى السجلات محكوم بميزة `audit_logs` والصلاحيات (super_admin يرى الكل، غيره حسب الشركة).

## ما يُسجّل حالياً

- **المصادقة (auth):** تسجيل الدخول (نجاح/فشل)، تغيير كلمة المرور، إعادة تعيين كلمة المرور، تسجيل الخروج، MFA (إعداد، تفعيل، تعطيل، تحقق).
- **المرضى (patient):** عرض (فتح ملف مريض)، إنشاء، تحديث، حذف.
- **المواعيد (appointment):** إنشاء، تحديث، حذف.
- **الفواتير (invoice):** إنشاء، دفع.
- **المصروفات (expense):** إنشاء، حذف.
- **مستندات المريض (patient_document):** رفع مستند.
- **الشركات (company):** إنشاء، تحديث.
- **الموظفون (staff):** إنشاء، تحديث، حذف.
- **الحجز الإلكتروني (online_booking):** تأكيد، إلغاء، إكمال.

يُنصح بمراجعة دورية لإضافة تسجيل لأي endpoint جديد يمس PHI (قراءة أو تعديل).

## سياسة الاحتفاظ (Retention)

- **المدة الافتراضية المقترحة:** الاحتفاظ بسجلات التدقيق لمدة **6 سنوات على الأقل** (و7 سنوات موصى بها) إذا كان النظام يخضع لمتطلبات حفظ سجلات الرعاية الصحية (مثل HIPAA أو أنظمة محلية).
- **التخزين:** السجلات مخزنة في MongoDB في مجموعة `audit_logs`. يُنصح بـ:
  - نسخ احتياطي دوري للمجموعة وفق سياسة النسخ الاحتياطي للمشروع.
  - تقييد صلاحيات الوصول إلى قاعدة البيانات بحيث لا يصل إلى `audit_logs` إلا الأدوار المصرح بها، ولا يُمنح صلاحية **تحديث أو حذف** للمجموعة للتطبيق أو للمستخدمين العاديين (السجلات للمراجعة فقط؛ لا تُحذف ولا تُعدّل).
- **عدم التعديل أو الحذف:** سجلات التدقيق تُعتبر سجلات ثابتة (append-only). التطبيق يضيف سجلات فقط؛ لا يوجد endpoint أو منطق لتعديل أو حذف سجلات `audit_logs`. صلاحيات قاعدة البيانات يجب أن تمنع التحديث/الحذف من أي مستخدم عادي.
- **الحذف (استثنائي):** إن رغبت في حذف سجلات أقدم من مدة معينة، يُنصح بتنفيذ عملية دورية (مثلاً job) تحذف الوثائق التي يتجاوز `timestamp` فيها المدة المحددة، مع توثيق السياسة وتوقيت التنفيذ، وعدم تجاوز الحد الأدنى 6 سنوات.
- **الوصول:** فقط المستخدمون الذين يملكون ميزة `audit_logs` (وفق الصلاحيات والعيادة) يمكنهم عرض السجلات عبر واجهة النظام أو API (`GET /api/audit-logs`).

## الحقول المسجلة في كل حدث

- `id`, `user_id`, `user_email`, `user_role`, `company_id`
- `action`, `resource_type`, `resource_id`, `details`
- `ip_address`, `user_agent`, `timestamp`

تعديل منطق التسجيل في [server.py](d:\TEBBI\TEBI--main\backend\server.py) (دالة `log_audit` واستدعاءاتها).
