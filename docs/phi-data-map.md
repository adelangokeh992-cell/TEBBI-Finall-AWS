# خريطة بيانات PHI (Protected Health Information) — TEBBI

للامتثال لـ HIPAA وتوثيق تدفق البيانات الصحية المحمية.

## 1. أماكن دخول PHI

| المصدر | الوصف | نوع البيانات |
|--------|--------|---------------|
| نماذج إضافة/تعديل مريض | واجهة المرضى (Patients) وواجهة الإداري | الاسم، الهوية، تاريخ الميلاد، الجنس، الهاتف، العنوان، الطوارئ، الحالات المزمنة، إلخ |
| استيراد (Backup restore) | استعادة نسخة احتياطية لعيادة | كل بيانات المرضى والمواعيد والفواتير والزيارات |
| API | `POST/PUT /api/patients`, `GET /api/patients`, إلخ | نفس حقول المريض والزيارات |
| بوابة المريض (Portal) | تسجيل الدخول وعرض المواعيد والرسائل | اسم المريض، بيانات الاتصال، المواعيد |
| الحجز العام (Public booking) | حجز موعد من الموقع العام | الاسم، الهاتف، البريد، بيانات الحجز |
| التقارير والطباعة | تقارير طبية، فواتير، وصفات | أسماء المرضى، التشخيصات، المبالغ |

## 2. أين تُخزَن PHI

| الموقع | المجموعات/الملفات | ملاحظات |
|--------|-------------------|----------|
| MongoDB (قاعدة البيانات الرئيسية) | `patients`, `visits`, `diagnoses`, `medications`, `allergies`, `medical_images`, `invoices`, `appointments`, `online_bookings`, `company_backups` (مشفّر), `patient_documents` | تشفير حقول حساسة عند الراحة (انظر [encryption.md](encryption.md)) |
| ملفات مرفوعة | إن وُجد تخزين ملفات (مثلاً مستندات مريض) | يجب أن تكون في مسار آمن وتشفير إن أمكن |

## 3. أين تُرسَل PHI (أو تُعالَج خارج التطبيق)

| الوجهة | الغرض | إجراءات |
|--------|--------|----------|
| سحابة الاستضافة (السيرفر الذي يشغّل التطبيق وقاعدة البيانات) | تشغيل التطبيق وقاعدة البيانات | استخدام مزود متوافق مع HIPAA وتوقيع BAA |
| خدمة البريد (إن وُجدت) | إرسال إشعارات أو روابط إعادة تعيين كلمة المرور | BAA إن كانت الخدمة تلمس محتوى صحي |
| OpenAI (إن مُفعّل) | تحليل أعراض، صور، نصوص تقارير | مراجعة سياسة الخصوصية واتفاقيات الاستخدام؛ عدم إرسال معرفات مرضى حقيقية إن أمكن |
| تحليلات (Analytics) | إن وُجدت أدوات تحليلات على الواجهة | عدم إرسال PHI؛ استخدام معرفات مجهولة أو إيقاف التتبع لصفحات تحتوي بيانات مرضى |
| النسخ الاحتياطية | نسخ يومية أو يدوية | النسخ مشفّرة؛ عند النسخ إلى موقع ثانٍ (S3، خادم آخر) التأكد من تشفير النقل والصلاحيات |

يُنصح بتحديث هذه الخريطة عند إضافة قنوات جديدة (نماذج، API، تكاملات، تخزين ملفات).
