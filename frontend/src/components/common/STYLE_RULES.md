# قواعد الأنماط الموحّدة (Tebbi UI)

## متى نستخدم كل مكوّن

- **PageHeader**: بداية كل صفحة داخلية — عنوان (h1) + وصف اختياري + أزرار إجراء + بادج حالة. استخدم `gradient={true}` للصفحات التي تريد لها شريط علوي مميز (مثل AI).
- **PageEmptyState**: عند عدم وجود بيانات في قائمة أو جدول — أيقونة + رسالة + زر إجراء اختياري (مثل "إضافة مريض").
- **StatCard**: بطاقة إحصائية صغيرة (عدد المرضى، الإيرادات، إلخ). استخدم في Dashboard وصفحات الملخص.
- **DataCard**: غلاف لأي قسم محتوى (جدول، قائمة، نموذج). استخدم `clickable` و `hoverBorderColor` للبطاقات القابلة للنقر.
- **LoadingSpinner**: مؤشر تحميل واحد موحد — استخدمه أثناء جلب البيانات.

## قواعد عامة

- **خلفية الصفحة**: الإبقاء على `bg-slate-50 dark:bg-slate-950` في الـ Layout؛ المحتوى داخل `main` بمسافات `space-y-6`.
- **الهيدر**: استخدم PageHeader أو نفس النمط (عنوان + وصف + أزرار) مع نفس الـ spacing والألوان.
- **البطاقات**: `rounded-xl` وحدود واضحة (`border-2` للتمييز). للقابلة للنقر: `hover:shadow-lg hover:scale-[1.02] transition-all` وألوان حدود عند الـ hover.
- **التبويبات (Tabs)**: `TabsList` و `TabsTrigger` بـ `rounded-xl`؛ `TabsContent` داخل حاوية ذات خلفية خفيفة وحدود عند الحاجة.
- **الجداول**: رؤوس وحدود موحّدة؛ صفوف مع `hover:bg-slate-50 dark:hover:bg-slate-800/50`.
- **الحالات الفارغة**: استخدم PageEmptyState بدلاً من نص بسيط.
- **التحميل**: استخدم LoadingSpinner في منتصف المنطقة المحمّلة.
- **RTL**: كل صفحة تأخذ `dir={language === 'ar' ? 'rtl' : 'ltr'}` على الحاوية الرئيسية؛ استخدم `me-`/`ms-` أو `pr`/`pl` بدلاً من `mr`/`ml`.
